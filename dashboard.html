Project Overview & Goals

British Columbia's public sector employs hundreds of thousands of people across health authorities, crown corporations, school districts, universities, municipalities, and government ministries. Every year, the province publishes salary data for all public servants earning above $75,000 — but the raw data, buried in PDFs and government portals, is nearly impossible for the average person to interrogate.

The goal of this project was to turn that raw transparency data into a genuinely useful analytical tool. Not just a searchable table, but a dashboard that answers real questions: Which sectors pay the most? Which organisations lead their peer group? What does a Nurse Practitioner at Fraser Health earn compared to one at Interior Health? Where does a given individual rank within their organisation?

Data Sources & Methodology

The primary data source is the Vancouver Sun's Public Sector Salary Database (PSSDB), which aggregates salary disclosure data from BC public sector organisations into a single cleaned dataset. The underlying data originates from BC government public accounts and individual organisation disclosures under the Financial Information Act.

Python Preprocessing Pipeline

Raw CSV data required several preprocessing steps before it could power the dashboard. A Python script handled all transformation, producing a precomputed JSON payload structured for fast browser rendering.

Step 1
Data Cleaning & Type Coercion
Remuneration column had mixed types (float and string) across rows — coerced to numeric with pd.to_numeric(errors='coerce'). 34 null salary rows dropped. Position nulls filled with "Unknown".
Step 2
Year Filtering
Dataset contains three year formats (2017/18, 2018, 2017) from different reporting periods. Filtered to 2017/18 as the primary fiscal year, yielding 59,858 clean records from 93,276 total rows.
Step 3
Aggregate Precomputation
All histogram bins, salary band counts, sector medians, organisation aggregates, and role aggregates precomputed in Python using pandas groupby and d3-compatible bin structures. Avoids heavy computation in the browser.
Step 4
Percentile Ranking
For each individual record, computed their salary percentile within their organisation by sorting org salary arrays and using bisection search. Each record stored with a precomputed percentile integer (0–100) for instant display in the search table.
Key Data Findings

The data reveals clear structural patterns in BC public sector pay — with health authorities dominating headcount, universities and crown corporations leading on median salary, and significant variation within sectors across individual organisations.

Finding 01
Health Authorities are the largest employer — but not the highest payer
With 28,767 disclosures, health authorities represent nearly half of all earners in the dataset. Yet median pay sits in the middle of the sector pack — driven down by the volume of nursing staff at lower salary bands.
Finding 02
Crown corporations show the widest salary range of any sector
Organisations like BC Hydro and Powerex (a subsidiary) contain both mid-level engineers and C-suite executives earning $800K+, creating extreme salary dispersion within a single classification.
Finding 03
Police departments punch above their weight on median salary
With only 3,003 disclosures, police departments have a compressed salary distribution — most officers cluster between $95K and $130K, yielding a relatively high median for the sector despite no executive-level outliers.
Finding 04
The $75K–$100K band contains nearly half of all earners
Despite the dataset covering only those earning above the disclosure threshold, the majority cluster in the first salary band — reflecting the large number of teachers, nurses, and government workers at or near the disclosure floor.

Technical Challenges & Solutions

Building a dashboard on 60,000 records in a single HTML file required solving several real engineering problems — not just visualisation design choices.

Challenge → Solution
Payload Size: 6MB+ data file
Embedding 60K raw records inline in HTML created an 11MB file with a multi-second blank screen on load. Solution: serve as a separate data.json file via GitHub Pages, which gzips automatically to ~1.2MB over the wire. The HTML fetches it asynchronously so the page skeleton renders immediately.
Challenge → Solution
9,262 Unique Role Titles
BC public sector position titles are completely unstandardised — "B1_Nurse Direct Patient Care level 3", "BCNU DC1 Nurse Upp", and "Registered Nurse Critical Care" are all distinct values for what is broadly the same role. The dashboard surfaces raw titles with minimum-headcount filtering rather than attempting normalisation, which would require ML classification or a manual taxonomy.
Challenge → Solution
Real-Time Search on 60K Records
Filtering 60,000 records on every keystroke risks UI freezes. Solution: precomputed compact search index with short key names (n, o, c, r, s, p) reduces memory footprint. Search uses simple Array.filter() with String.includes() — fast enough at 60K rows without needing a search index library.
Challenge → Solution
Mixed Data Types in Source CSV
The Remuneration column contained mixed string/float types across rows, causing pandas DtypeWarning and silent NaN coercion. Fixed with explicit low_memory=False and pd.to_numeric(errors='coerce') followed by null-row removal.
"The hardest part wasn't building the charts — it was making 60,000 rows feel fast."
